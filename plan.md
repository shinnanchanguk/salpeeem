# 살핌 (Salpeem) — 프로젝트 플랜

## 프로젝트 개요
- 한줄 설명: 교사가 수업 중 관찰을 30초 안에 메모하면, AI가 생활기록부 문장으로 변환하고 학생별·영역별로 축적하여 학기말 생기부 작성을 완성하는 데스크톱 앱
- 타겟 유저: 한국 중·고등학교 교사 (담임·교과 교사)
- 핵심 가치 제안: "보는 순간, 기록이 됩니다" — 수업 흐름을 끊지 않는 30초 기록 → 학기말 생기부 완성까지 하나의 워크플로우

## 기능 명세

### 핵심 기능
1. **빠른 기록 입력**: 오타·줄임말 그대로 입력 → AI가 생기부 문장으로 변환. @학생 /영역 태그로 자동 분류. 태그 없으면 인박스로
2. **커스텀 뷰 (스마트 폴더)**: 학생×영역×학급을 자유롭게 조합하여 이름 붙인 뷰 생성. 출처(기록/과제/설문) 무관하게 해당하는 문장을 한곳에 표시
3. **과제·설문 문장화**: 과제 안내사항 + 학생 제출물 업로드 → 학생별 생기부 문장 자동 생성. 설문 응답도 동일
4. **문장 완성하기 (학기말)**: 학생·영역 선택 → 축적된 문장 기반 일괄 초안 생성 → 개별 AI 대화로 다듬기 → 최종 확정
5. **영역 완성 (개별 상세)**: 특정 학생×영역의 축적 문장 확인 + 중요도 조절 + AI 대화로 초안 완성. 바이트 제한 내 생성
6. **세 가지 모드**: 전체(풀 기능) / 사이드(세로 패널) / 바(입력란 한 줄). 글로벌 단축키로 즉시 전환
7. **온보딩**: 첫 실행 시 5스텝 튜토리얼 (인트로 → 빠른 기록 → 커스텀 뷰 → 세 가지 모드 → 시작)

## 화면 구조

### 공통 레이아웃: 3-layer 구조
레퍼런스 디자인(폴더 탭 노트앱)에서 차용한 구조:
- **Layer 1 — 상단 폴더 탭**: 기록 / 과제·설문 / 문장 완성하기 / 설정
- **Layer 2 — 좌측 패널**: 탭마다 맥락이 다른 리스트/선택 패널
- **Layer 3 — 우측 메인 영역**: 선택에 따라 바뀌는 작업 공간

### 화면별 상세

| 화면 | 상단 탭 | 좌측 패널 | 우측 메인 | 디자인 |
|------|---------|-----------|-----------|--------|
| 메인 기록 | 기록 (활성) | 인박스(고정, 첫 번째) + 커스텀 뷰 목록 + "새 뷰 만들기" | 입력란(항상 고정) + 하단(인박스 또는 뷰 콘텐츠) | ✅ Variant |
| 과제·설문 | 과제·설문 (활성) | 등록된 과제/설문 리스트 | 등록 폼 또는 결과 화면 | ✅ Variant |
| 문장 완성하기 | 문장 완성하기 (활성) | 학생 트리 + 영역 체크 선택 패널 | 영역별/학생별 뷰 + 일괄 초안 생성 | ✅ Variant |
| 영역 완성 | 문장 완성하기 (활성, 드릴다운) | — | 좌: 축적 문장 + 중요도 + 바이트 바 / 우: AI 대화 + 초안 | ✅ Variant |
| 설정 | 설정 (활성) | 설정 카테고리 리스트 | 선택된 카테고리 상세 | ✅ Variant |
| 온보딩 | — (모달 오버레이) | — | 5스텝 시퀀스 | ❌ 토큰 기반 |

### 핵심 UX 결정

**빠른 기록 입력란은 항상 고정**
좌측 패널에서 어떤 뷰를 선택하든 우측 상단의 입력란은 절대 사라지지 않음. 커스텀 뷰를 보면서도 바로 기록 가능.

**커스텀 뷰 (스마트 폴더)**
- 학생(개인/학급/학년) × 영역을 자유롭게 조합하여 뷰 생성 + 이름 부여
- 출처(직접 기록, 과제, 설문) 무관하게 해당하는 모든 문장이 한곳에 표시
- 좌측 패널 첫 번째 항목은 항상 "인박스" (자동, 삭제 불가)

**영역 완성은 문장 완성하기의 하위 뷰**
별도 탭이 아님. 문장 완성하기 탭에서 특정 학생×영역 클릭 시 우측이 드릴다운.

**바이트 제한 + 중요도 관계**
- 각 영역에 바이트 제한이 설정됨 (설정 > 영역 관리에서)
- 영역 완성 화면에서도 제한값 조정 가능 (인라인 편집)
- 문장별 중요도(높음/보통/낮음)를 교사가 설정
- 중요도가 '낮음'인 문장은 AI가 초안 생성 시 축약하거나 생략
- AI 초안 박스 근처에도 실시간 바이트 카운터 표시

## 기술 결정
- 프레임워크: **Tauri v2** (데스크톱 앱)
- 프론트엔드: **Vite + React + TypeScript**
- 스타일링: **Tailwind CSS + shadcn/ui**
- 상태관리: **Zustand**
- 애니메이션: **Framer Motion**
- 로컬 DB: **SQLite** (Tauri SQL 플러그인)
- AI 연결: **OpenRouter API** (기본 사용량 제공 + 사용자 직접 API 키 연결 옵션)
- 데이터 저장: 전부 로컬 (서버 없음)
- 개인정보 보호: 학생 개인정보는 AI에 전송하지 않음. 문장만 보내고 받음

### Tauri 활용 기능
- 윈도우 모드 전환 (전체/사이드/바): Tauri Window API
- 글로벌 단축키: Tauri Global Shortcut API
- 파일 업로드 다이얼로그: Tauri File Dialog API
- 로컬 파일 시스템 접근: Tauri FS API
- 데이터 내보내기/가져오기: Tauri FS API

## 데이터 모델

### students (학생)
- id, name, grade(학년), class(반)

### categories (영역)
- id, name, byte_limit

### records (기록)
- id, raw_input(원문), generated_sentence(변환 문장), student_id(nullable), category_id(nullable), source(기록/과제/설문), importance(높음/보통/낮음), created_at
- student_id, category_id가 null이면 인박스 상태

### custom_views (커스텀 뷰)
- id, name, student_filter(학생/반/학년 조합, JSON), category_filter(영역 조합, JSON)

### assignments (과제)
- id, title, category_id, instructions, created_at

### surveys (설문)
- id, title, instructions, created_at

### completed_records (완성된 생기부)
- id, student_id, category_id, final_text, byte_count, status(미완성/완성됨), confirmed_at

## 사용자 플로우

### 일상 (수업 중)
1. 앱 열기 (또는 글로벌 단축키로 바/사이드 모드 전환)
2. 관찰 메모 입력 (@학생 /영역 태그)
3. AI가 즉시 생기부 문장으로 변환 → 자동 분류
4. 태그 안 붙인 건 인박스 → 나중에 정리

### 과제·설문 (수시)
1. 과제·설문 탭 → 영역 선택 + 안내사항 입력
2. 학생 제출물/응답 업로드
3. AI가 학생별 문장 생성 → 확인·수정 → 확정
4. 확정된 문장은 해당 학생×영역으로 자동 분배

### 학기말 (생기부 작성)
1. 문장 완성하기 탭 → 학생·영역 선택
2. 일괄 초안 생성 (목표 바이트 설정)
3. 개별 학생×영역 클릭 → 영역 완성 진입
4. 축적 문장 중요도 조정 → AI 대화로 초안 다듬기
5. 바이트 제한 내 최종 확정

## 제약 사항 / 합의 사항
- 온보딩은 Variant 디자인 없이 기존 토큰 기반으로 Claude Code가 구현
- 디자인 톤: Variant 원본 코드에 확정되어 있음. 별도 해석하지 말 것.
- 레퍼런스: 폴더 탭 노트앱 디자인 — 상단 폴더형 탭 + 좌측 리스트 + 우측 작업 영역의 3-layer 구조
- 데이터는 전부 로컬 저장 (서버 없음)
- 학생 개인정보 AI 미전송 원칙
- Next.js가 아닌 Vite + React 사용 (Tauri 데스크톱 앱이라 SSR 불필요)
