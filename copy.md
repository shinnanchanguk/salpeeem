# 살핌 (Salpeem) — UI Copy

---

## 상단 탭 네비게이션

- 로고: 살핌
- 탭 1: 기록 (기본 활성)
- 탭 2: 과제·설문
- 탭 3: 문장 완성하기
- 탭 4: 설정

---

## 1. 온보딩 (첫 실행)

### 스텝 1: 인트로
- 헤드라인: 보는 순간, 기록이 됩니다.
- 서브: 학생의 모든 성장을 놓치지 않고 기록하세요.
- 본문: 수업 중 떠오른 관찰을 30초 안에 입력하면, 살핌이 생기부 문장으로 바꿔서 학생별·영역별로 쌓아둡니다. 학기말엔 쌓인 문장으로 영역 생기부를 완성하세요.

### 스텝 2: 빠른 기록 사용법
- 헤드라인: 떠오른 순간 바로 적으세요
- 안내 1: @학생이름 — 학생을 태그합니다
- 안내 2: /영역이름 — 영역을 태그합니다
- 안내 3: 태그 없이 적어도 괜찮아요. 인박스에 쌓이고, 나중에 정리할 수 있습니다.
- 예시 1 입력: "@김도윤 피타고라스 문제 푸는데 기하적 설명 지오지브라 써서 해가지고 창의적이엇고 반응 좋았음"
- 예시 1 변환 결과: "피타고라스 문제를 해결하는 과정에서 공학적 도구를 활용하여 문제의 기하적 상황을 시각화 하는 창의력을 발휘함."
- 예시 2 입력: "@이서연 학급허ㅣ의에서  Canva로 PPT 만들어서 친구들 사전 설문 통계낸거 시각화해서 보여주느넫 반응 좋았음"
- 예시 2 변환 결과: "학급자치회의에서 사전 설문 조사 결과를 시각화하여 통계 데이터로 보여주는 프레젠테이션을 진행하며 급우들의 큰 호응을 얻음."
- 예시 안내: 오타, 줄임말 그대로 적어도 됩니다. 살핌이 생기부 문장으로 다듬어줍니다.

### 스텝 3: 나만의 뷰 만들기
- 헤드라인: 보고 싶은 기록만 모아보세요
- 본문: 학생, 영역, 학급을 자유롭게 조합해서 나만의 뷰를 만들 수 있습니다. 직접 기록, 과제, 설문 — 출처와 상관없이 해당하는 문장이 한곳에 모입니다.
- 예시 1: "1반 세특" — 1반 학생 전체 × 세특 영역
- 예시 2: "김도윤 전체" — 김도윤 학생의 모든 영역
- 예시 3: "2학년 행특" — 2학년 전체 × 행특 영역
- 안내: 좌측 패널에서 뷰를 추가하고, 클릭하면 오른쪽에 해당 기록이 표시됩니다.

### 스텝 4: 세 가지 모드
- 헤드라인: 상황에 맞게 창을 바꾸세요
- 모드 1 이름: 전체
- 모드 1 설명: 기록, 학생 관리, 영역 완성까지 모든 기능
- 모드 2 이름: 사이드
- 모드 2 설명: 다른 작업 옆에 세로로 붙여두고 기록
- 모드 3 이름: 바
- 모드 3 설명: 화면 위아래에 입력란 한 줄만. 수업 중 빠르게.
- 안내: 단축키로 즉시 전환됩니다. 설정에서 단축키를 변경할 수 있어요.

### 스텝 5: 시작
- 버튼: 시작하기
- 하단 안내: 학생 명단부터 등록하면 @태그가 바로 작동합니다.

## 2. 메인 — 기록

### 좌측 패널 (커스텀 뷰)
- 첫 번째 항목: 인박스 (자동 생성, 삭제 불가)
- 그 아래: 사용자가 생성한 커스텀 뷰 목록
- 각 뷰 항목: [뷰 이름] — [문장 N건]
- 하단 버튼: + 새 뷰 만들기
- 빈 상태 (커스텀 뷰 없을 때): 자주 보는 학생·영역 조합으로 뷰를 만들어보세요

### 뷰 생성
- 다이얼로그 제목: 새 뷰 만들기
- 이름 입력 라벨: 뷰 이름
- 이름 입력 플레이스홀더: 예: 1반 세특, 김도윤 전체
- 학생 선택 라벨: 학생
- 학생 선택 구조: 학년 > 반 > 학생 (트리 형태, 학년·반 단위 전체 선택 가능)
- 학생 검색: "이름으로 검색"
- 영역 선택 라벨: 영역
- 영역 선택 구조: 등록된 영역 목록 체크 (전체 선택 가능)
- 생성 버튼: 만들기
- 생성 완료 토스트: "[뷰 이름] 뷰가 생성되었습니다"

### 우측 메인 영역

#### 빠른 기록 입력란 (항상 고정)
- 플레이스홀더: "관찰을 적으세요. @학생 /영역으로 바로 분류됩니다"
- 전송 버튼: 기록
- 위치: 좌측에서 어떤 뷰를 선택하든 항상 우측 상단에 고정

#### 자동완성
- @입력 시: 학생 이름 자동완성 드롭다운
- /입력 시: 영역 이름 자동완성 드롭다운

#### 인박스 선택 시 (기본 상태)
- 섹션 라벨: 인박스
- 설명: 학생 또는 영역이 지정되지 않은 기록
- 빈 상태: 모든 기록이 분류되었습니다
- 각 항목 구성: [날짜·시간] — [입력 내용] — [생성된 문장]
- 각 항목 액션: 학생·영역 지정하기
- 항목 지정 완료 시 토스트: "[학생명] / [영역명]으로 이동했습니다"

#### 커스텀 뷰 선택 시
- 섹션 라벨: [뷰 이름]
- 각 항목 구성: [날짜] [출처 태그: 기록 / 과제 / 설문] — [학생명] — [생성된 문장]
- 항목 클릭 시: 원문과 변환 결과 상세 펼침
- 빈 상태: 이 조합에 해당하는 기록이 아직 없습니다

## 3. 과제·설문

### 상단
- 페이지 제목: 과제·설문
- 설명: 안내사항과 학생 제출물을 올리면, 학생별 생기부 문장으로 변환됩니다.

### 과제 등록
- 영역 선택 라벨: 영역
- 영역 선택 플레이스홀더: 이 과제가 들어갈 영역을 선택하세요
- 안내사항 입력 라벨: 과제 안내사항
- 안내사항 플레이스홀더: "어떤 과제인지 적어주세요. AI가 이 맥락을 보고 문장을 만듭니다."
- 제출물 업로드 라벨: 학생 제출물
- 업로드 안내: 파일을 드래그하거나 클릭해서 업로드. 파일명에 학생 이름을 포함하면 자동 매칭됩니다.
- 변환 버튼: 문장 생성하기

### 설문 등록
- 설문 안내사항 라벨: 설문 안내사항
- 설문 안내사항 플레이스홀더: "어떤 설문인지 적어주세요."
- 응답 업로드 라벨: 학생 응답
- 업로드 안내: CSV, 엑셀, 또는 개별 파일로 업로드
- 변환 버튼: 문장 생성하기

### 결과 화면
- 결과 라벨: 생성된 문장
- 각 항목 구성: [학생명] — [생성된 생기부 문장] — [수정] [확정]
- 전체 확정 버튼: 전체 확정 — 각 학생 영역으로 분배
- 확정 후 토스트: "[N]명의 문장이 각 학생 영역으로 이동했습니다"

## 4. 문장 완성하기

### 상단
- 페이지 제목: 문장 완성하기
- 설명: 학생과 영역을 선택하면, 축적된 문장을 바탕으로 생기부를 완성합니다.

### 선택 패널

#### 학생 선택
- 라벨: 학생
- 구조: 학년 > 반 > 학생 (트리 형태로 펼쳐서 체크)
- 전체 선택: 학년 단위, 반 단위 전체 선택 가능
- 검색: "이름으로 검색"

#### 영역 선택
- 라벨: 영역
- 구조: 등록된 영역 목록 체크 (세특, 행특, 진로, 동아리, 봉사 등)
- 전체 선택 가능

### 결과 뷰 (선택 완료 후)

#### 뷰 자동 전환 규칙
- 선택된 영역 수 ≤ 선택된 학생 수 → 영역별 뷰 (기본)
- 선택된 학생 수 < 선택된 영역 수 → 학생별 뷰 (기본)
- 수동 전환 토글: 영역별 보기 / 학생별 보기

#### 영역별 뷰
- 각 영역 섹션: [영역명] — 완성 [M]명 / 선택 [T]명
- 영역 클릭 시 펼침: 해당 영역의 선택된 학생 리스트
- 각 학생 행: [학생명] — [반] — 문장 [N]건 — [완성됨 / 미완성]
- 학생 클릭 시: 개별 영역 완성 진입

#### 학생별 뷰
- 각 학생 섹션: [학생명] — [반]
- 학생 클릭 시 펼침: 해당 학생의 선택된 영역 리스트
- 각 영역 행: [영역명] — 문장 [N]건 — [완성됨 / 미완성]
- 영역 클릭 시: 개별 영역 완성 진입

### 일괄 초안 생성
- 버튼: 전체 초안 생성
- 바이트 설정 라벨: 목표 바이트
- 바이트 설정 안내: 각 학생의 초안이 이 바이트 이내로 생성됩니다
- 대상 선택: 전체 / 미완성만
- 생성 중 상태: [N]명 중 [M]명 생성 완료…
- 생성 완료 토스트: "[N]명의 초안이 생성되었습니다. 학생별로 확인하고 수정하세요."

### 일괄 생성 결과
- 각 학생 행: [학생명] — [영역명] — [초안 미리보기 1줄] — [바이트] — [수정] [확정]
- 수정 클릭 시: 개별 영역 완성 AI 대화 진입
- 전체 확정 버튼: 전체 확정
- 전체 확정 토스트: "[N]건의 생기부가 확정되었습니다"

## 5. 영역 완성 (개별)

### 상단
- 제목: [학생명] — [영역명] 완성하기

### 문장 취합
- 라벨: 축적된 문장
- 각 문장: [날짜] [출처 태그: 기록 / 과제 / 설문] — [문장 내용]
- 각 문장에 중요도: 높음 / 보통 / 낮음
- 바이트 현황: 현재 [N]바이트 / 제한 [M]바이트
- 바이트 초과 시 안내: 제한을 초과했습니다. 중요도가 '낮음'인 문장은 AI가 축약하거나 생략합니다.

### AI 대화
- AI 대화 영역 라벨: AI와 함께 완성
- AI 초기 메시지: "[학생명]의 [영역명] 초안을 만들었습니다. 수정하고 싶은 부분을 말씀해주세요."
- 초안 박스 근처 실시간 바이트 카운터: 현재 [N]바이트 / 제한 [M]바이트
- 입력 플레이스홀더: "이 학생의 리더십을 더 부각해줘", "두 번째 문장을 구체적으로" 등
- AI 컨텍스트 안내: AI는 이 학생의 모든 기록, 과제, 설문 문장을 기억하고 있습니다.

### 최종 확정
- 확정 버튼: 최종 확정
- 확정 후 토스트: "[학생명]의 [영역명] 생기부가 완성되었습니다"

## 6. 설정

### 학생 명단
- 라벨: 학생 명단 관리
- 안내: 엑셀 업로드 또는 직접 추가. 학년·반·이름 포함.
- 업로드 버튼: 명단 업로드 (엑셀/CSV)
- 직접 추가: + 학생 추가
- 각 항목: [이름] — [학년] — [반] — [수정] [삭제]

### 영역 관리
- 라벨: 영역 관리
- 안내: 생기부 영역을 등록하세요. 기록과 과제가 이 영역으로 분류됩니다.
- 기본 영역 예시: 세특, 행특, 진로, 동아리, 봉사
- 각 항목: [영역명] — [바이트 제한] — [수정] [삭제]
- 추가: + 영역 추가

### AI 연결
- 라벨: AI 연결
- 기본 상태 안내: 기본 사용량이 제공됩니다. 추가 사용이 필요하면 직접 API 키를 연결하세요.
- 사용량 표시: 이번 달 사용량 [N] / [M]건
- 토글: 내 API 키 사용
- 토글 ON 시:
  - 입력 라벨: OpenRouter API 키
  - 플레이스홀더: sk-or-...
  - 모델 선택 라벨: 모델
  - 모델 안내: 성능과 비용을 고려해 선택하세요
- 개인정보 안내: 학생 개인정보는 전송되지 않습니다. 문장만 보내고 받습니다.

### 단축키
- 라벨: 단축키 설정
- 전체 모드: [키 조합]
- 사이드 모드: [키 조합]
- 바 모드: [키 조합]
- 빠른 기록 포커스: [키 조합]

### 데이터
- 라벨: 데이터 관리
- 백업: 데이터 내보내기 (파일로 저장)
- 복원: 데이터 가져오기
- 안내: 모든 데이터는 이 기기에만 저장됩니다.
